about graft

graft is a static file preprocessor made in go

why is it useful

imagine this, youre making a website, you need to make the 50th page of your blog, so you duplicate your previous blog posts html file. after finishing your 50th blog post, you realize that its time for a new header, but... oh no! you have to edit every single file and its header!

guess what you can do with graft. its basically a non-runtime templating engine so you can just have a header.html then in your base.html file you can include your header.html like <g-include src="header.html" /> and voila! all your pages include only a reference to your header instead of a complete copy of it.

you can also use the extend command to do templating, you can have a base.html with a bunch of <g-block name="[name]" /> commands and then in your index.html file have a <g-extend src="base.html"> command with <g-define name="[name]"> commands and voila again! all your pages all have the same underlying html structure

why use graft instead of a templating engine

  • any language. it works on any text. its not html only, markdown only, etc.
  • no runtime. everything is processed at build time. it only outputs plain files and theres no dependancies after its built
  • user commands. many preprocessors hardcode behavior, but graft commands are go code and written by the user. you can build anything into graft you want
  • tree based. since commands rewrite an AST, it makes it easy to build complex transformations and extensions
  • very hackable. there is a preset package which will let you get quickstarted but if you need more customization, you can drop to a lower level and deal with the tokenizer and parser yourself
  • very lightweight. other templating engines can be very heavy, especially since many are runtime based. which means a worse experience for users because of website loading times
